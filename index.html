Framing: what analysts actually do

Analysts in industry and research don’t get rewarded for “writing code”. The value their role creates also lies in producing reliable insight and communicating it clearly. To do this we must show which steps we take and the assumptions we have undertaken, it is important that our logic is justified and can be followed by others.

The lifecycle

Question → Data → Transformation → Insight → Communication

Code is the tool that makes this process repeatable and shareable.

RStudio basics

What is R, and why use it?

  • R is a programming language for data analysis and statistics.
  • It’s used heavily in research and in industry (data science, health, finance, etc.). There is also a growing demand for it in the modelling space.
  • It is powerful and popular because it is free, open-source and has an active community for support and development. There is also extensive documentation supporting all of the packages so that you can find out how to use them to their fullest extent.
  • After you write an analysis once, you can re-run it on new data quickly and consistently.

RStudio layout (what you’re looking at)

Script (top-left)

Where you write and save code. If it matters and you want to keep a hold of it to re-run in future, it goes here.

Environment (top-right)

Shows objects currently loaded (datasets, variables, functions).

Console (bottom-left)

Where code runs and you can see outputs. It is also good for one-off testing of lines/chunks of code, but don’t rely on it for your final work.

Files / Plots / Help (bottom-right)

Navigate folders, view plots and outputs, and read documentation.

Projects

A project is one folder containing everything for one piece of work: data, scripts, outputs, and notes. Having an organised project folder prevents broken paths and lost files - trust me... life is easier when everything is where it should be.

Interrogate the dataset: what does it claim to represent?

Run these in your script, and read the outputs carefully:

head(df)
glimpse(df)
summary(df)

# Missing values per column
colSums(is.na(df))
Task: dataset story (write answers as comments in your script)
  • What does one row represent?
  • What does each column represent (units / meaning)?
  • Which columns are numeric vs categorical?
  • Are there missing values? Where?
  • What questions could this dataset help answer?
Comment style example
# one row seems to represent ... e.g. the PK data for a given patient at one specific timepoint
# Main columns are ID, TIME, Conc and likely represent XYZ
# Potential questions: (1) X (2) Y
